# 📝 日記カケイボ

日記を書く"ついで"に、家計簿もつけられるWebアプリです。

家計簿だけを単体でつけようとすると面倒で続かない人でも、もともと習慣化している「日記を書く」という行為の延長線上でお金の記録が残るようにする、という体験の変化を狙っています。

🔗 **本番サイト**: https://web3ai-week10.vercel.app

---

## ✨ できること

### 1. 日記を書くだけでAIが支出・収入を自動検出
自由な文章で日記を書いて「✨ AIに分析してもらう」を押すと、OpenAI APIが文章の中からお金が動いた一文だけを抜き出し、支出/収入・カテゴリ・おおよその金額を提案します。

- 検出結果はカテゴリタグ付きで一覧表示され、不要なものは「🚫 除外する」でその場から除外可能
- AIが見逃した文があれば「検出されなかった文」から手動で追加もできる

### 2. チャット形式で金額・支払い方法を確認
検出された項目を1件ずつ、チャットのようなUIで「いくらでしたか？」「何で払いましたか？」と確認しながら記録していきます。

### 3. 残高不足時の補填提案
支払い方法（現金/PayPay/銀行）の残高が足りない場合、自動で不足額を検知し「どこから補填しますか？」と他の口座からの付け替えを提案します。

### 4. 月ごとにグループ化された日記一覧
書いた日記は「2025年7月（3件）」のように月ごとにまとめて表示され、見出しをタップすると折りたたみ／展開ができます。

### 5. 支出・収入それぞれの推移グラフ
ダッシュボードには
- カテゴリ別の今月の支出（棒グラフ）
- 支出推移（月別合計 / カテゴリ別の折れ線グラフ、直近12ヶ月）
- 収入推移（支出グラフと同じ軸・単位・粒度の折れ線グラフ）

が表示され、家計の流れを一目で把握できます。

### 6. 初期残高の設定・データリセット
現金・PayPay・銀行それぞれの初期残高を設定でき、全データのリセットも可能です。

## 🛠 技術構成

| 項目 | 内容 |
|---|---|
| フロントエンド | 素の HTML / CSS / JavaScript（フレームワークなし、単一の `index.html`） |
| AI連携 | OpenAI API（`gpt-4o-mini`）による自然文からの支出・収入抽出 |
| サーバーサイド | Vercel Serverless Function（`api/analyze.js`） |
| データ保存 | ブラウザの `localStorage`（サーバー側にユーザーデータは保存しない） |
| ホスティング | Vercel |
| フォント | Fredoka（見出し・ボタン） / Zen Maru Gothic（本文） |

### なぜOpenAI + サーバーレス関数という構成にしたか
日記の文章から「お金が動いたこと」を読み取る部分だけはLLMの言語理解に任せ、金額の集計・残高管理・グラフ描画など「決まった手順を正確にこなす仕事」は自前のJavaScriptロジックで行う、という役割分担にしています。

またAPIキーをブラウザに直接埋め込むと誰でも閲覧・盗用できてしまうため、キーは **Vercelのサーバー環境変数にのみ保存し、ブラウザは自分のサーバー（`/api/analyze`）を呼ぶだけ** という設計にしています。これにより利用者側にAPIキーが一切露出しません。

```
ブラウザ（index.html）
   │  fetch('/api/analyze', { text: 日記の本文 })
   ▼
Vercel Serverless Function（api/analyze.js）
   │  process.env.OPENAI_API_KEY を使って呼び出し
   ▼
OpenAI API（gpt-4o-mini）
```

## 🚀 セットアップ・デプロイ方法

### ローカルで動かす（AI機能なし・見た目確認用）
```bash
cd web3ai_week10
python3 -m http.server 8000
# http://localhost:8000 にアクセス
```
静的サーバーだけでは `/api/analyze` が動かないため、AI検出機能は404になります。

### ローカルでAI機能も含めて動かす
```bash
npm i -g vercel
cd web3ai_week10
vercel dev
```

### 本番デプロイ
1. Vercelプロジェクトの環境変数に `OPENAI_API_KEY` を設定（Production）
   ```bash
   vercel env add OPENAI_API_KEY production
   ```
2. デプロイ
   ```bash
   vercel --prod
   ```

---

## 🔒 セキュリティについて
- OpenAIのAPIキーはVercelの暗号化された環境変数としてのみ保持し、コードやリポジトリ、ブラウザのレスポンス・ネットワークログには一切含まれません。
- ユーザーの日記・家計簿データはすべてブラウザの `localStorage` に保存され、サーバーには送信・保存されません（分析リクエスト時に日記の本文がOpenAI APIに送られる点のみ例外です）。
